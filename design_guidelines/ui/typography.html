Page guideline · Typography

Two families. One voice.

Plus Jakarta Sans carries every hero and everything below it — display at weight 600, body at 400. JetBrains Mono handles labels, captions, and code. No serif, no italic: emphasis is one coral word at weight 600. This page documents the full scale so every CocoIndex page — homepage, docs, examples, blog — sounds the same.

03 — Docs scale

A tighter scale for docs.

Technical documentation is scanned, not read. The docs site runs a sidebar + content + TOC layout that squeezes the content column to ~720px, and H3s appear several times per page. A denser type scale — fixed sizes, tighter hierarchy, no clamp — serves readers better. This variant is the ratified default for the docs site only; marketing, blog, and examples keep the canonical Scale in §02.

Do

Use the docs scale inside /docs.

15.5 / 28 / 18 applies to every reference page. The denser hierarchy preserves the h1(60)→h2(28)→h3(18) ratio at the narrower column width — the marketing 60→44→22 would flatten visibly.

Do

Lift from Stripe, Vercel, GitHub.

14–16px body is the standard for scan-heavy technical documentation. 15.5 lands in the middle of the pack — dense enough to reference, comfortable enough to read inline.

Don't

Don't mix scales inside docs.

Once a page enters the docs shell, every heading and paragraph uses this scale. Pulling a 17px paragraph or a 44px H2 in from a marketing template breaks the rhythm the reader has already calibrated to.

Don't

Don't apply this scale outside docs.

Marketing, blog, examples, and the home page all read paragraph-by-paragraph in wider columns. Those surfaces keep §02 — Scale. The docs variant exists because the shell is narrow, not because the brand got denser.

04 — Rules

How these work together.

Four rules keep the system legible across every page.

Do

Display weight is for the hero, nothing else.

Plus Jakarta Sans at weight 600 with −0.025em tracking carries the hero H1. Section heads, cards, and body use the same family at lighter weights. This keeps the display weight rare enough to stay powerful.

Do

Emphasis is coral, weight 600.

Inside a hero, wrap the one word you want to sing in <em>. The em gets coral + weight 600. Never italic, never underlined, never more than one word per hero.

Do

Use clamp() for every hero and H2.

Fluid sizing prevents awkward tablet breakpoints. Body text and labels stay at fixed px — only headings respond to viewport width.

Do

Caps labels always use mono.

Any uppercase text with 0.14em tracking — eyebrows, metadata, tags, sidebar section heads — is JetBrains Mono, 11px. Never letter-spaced Plus Jakarta Sans uppercase.

Don't

Don't mix weights inside a single hero.

The hero is display weight 600 only. No 400, no 700, no italics. If you need more emphasis, change the copy — not the type.

Don't

Don't introduce a third family.

No serif, no italic, no "friendly" handwritten scripts, no additional monospace. Plus Jakarta Sans + JetBrains Mono is the whole system.

05 — Tokens

Drop these into :root.

The full CSS custom-property set for typography.

/* font families */
--sans:  "Plus Jakarta Sans", -apple-system, BlinkMacSystemFont, "Helvetica Neue", Helvetica, Arial, sans-serif;
--mono:  "JetBrains Mono", ui-monospace, Menlo, monospace;

/* hero sizes */
--fs-display: clamp(56px, 7.4vw, 112px);  /* homepage only */
--fs-hero:    clamp(40px, 4.6vw, 60px);    /* every other page */
--lh-hero:    0.98;
--tr-hero:    -0.03em;

/* section + subsection */
--fs-h2: clamp(28px, 3.2vw, 44px);
--fs-h3: 22px;
--lh-h2: 1.05;
--lh-h3: 1.2;

/* body + lede */
--fs-lede: clamp(17px, 1.4vw, 20px);
--fs-body: 17px;
--lh-body: 1.55;

/* labels + code */
--fs-caps:    11px;
--fs-caption: 12px;
--fs-code:    13px;
--tr-caps:    0.14em;

/* docs scale — override inside /docs only (§03) */
--fs-docs-body: 15.5px;
--lh-docs-body: 1.65;
--fs-docs-h2:   28px;
--fs-docs-h3:   18px;
