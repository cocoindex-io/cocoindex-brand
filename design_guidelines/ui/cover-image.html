Page guideline · Blog cover image

One canvas. 1.91:1. Every share.

Every blog post ships a single cover.png mastered at 2400 × 1260. That's the Open Graph standard ratio — Facebook, LinkedIn, Slack, Discord, iMessage, Twitter/X, Bluesky, Hashnode, Substack and WordPress all expect 1.91:1. One file, every preview, no crop.

01 — Canvas

Master at 2400 × 1260.

Width is 2× the 1200 px Open Graph minimum so retina displays render every glyph crisply. Anything under 1200 px wide downgrades to a thumbnail card on Facebook and LinkedIn.

PropertyValueNotes
Aspect ratio 1.91 : 1 Open Graph spec. Adopted by every major link-preview surface.
Master size 2400 × 1260 px 2× retina. Sharp on DPR 2 displays.
Minimum size 1200 × 630 px Below this, social previews render the small thumbnail card, not the wide card.
Maximum size ≤ 8000 px on long edge Twitter and WhatsApp re-encode aggressively past 5 MB.
File format PNG (or JPG for photographic) Astro emits responsive variants automatically. PNG keeps text edges sharp.
File size target < 800 KB PNG · < 200 KB JPG/WebP Keeps LCP under 2.5 s on 4G.
Filename cover.png Sits next to index.md. Referenced via heroImage: ./cover.png frontmatter. 
# content collection schema (src/content/blogs/<slug>/index.md)
heroImage: ./cover.png     # required · 2400×1260
heroAlt:   "<descriptive title>"

Safe area

Twitter occasionally crops 1.91:1 down to 2:1 in-feed. Keep the title, mascot, and any critical detail inside the center 88% of the canvas. The dashed coral box below is the safe area; the dotted line is the typical Twitter crop.

In-page frame · consistent padding

When the cover renders inside the blog post, the surrounding .cover-art container is the visual frame: aspect-ratio matches the source (1.91 : 1) so the image never crops, and a 14 px cream inset with a hairline rule gives every cover the same breathing room. The image inside uses object-fit: contain — letterbox before crop. Any cover that doesn't ship at 1.91 : 1 will sit on cream bars; do not rebalance it by switching to cover.

/* src/styles/globals.css · .blog-theme .cover-art */
aspect-ratio: 1.91 / 1;     # matches the source canvas — no crop
background:   var(--cream);
border:       1px solid var(--rule);
padding:      14px;            # consistent inset on every post
border-radius: 12px;
/* img */
object-fit:   contain;         # letterbox, never crop
border-radius: 8px;

D · corner mark Brand sigil. Always present. JetBrains Mono · 16 px · uppercase · letter-spacing 0.18em · coral dot + "CocoIndex · <Section>".
E · art Central illustration: code, diagram, or mascot scene. Sits inside the safe area. Background contrast must clear AA on cream. No drop shadows.

Mascot · canonical SVG only

If Coco appears on a cover, the mascot must be the canonical brand SVG — never a redrawn, simplified, or AI-generated copy. Inline the file at /design_guidelines/assets/cocoindex.svg (mirror at /public/images/cocoindex.svg) so the renderer ships an offline-safe PNG. Re-drawing the coconut shell, eyes, or palm leaves drifts the brand silhouette every release; lock it.

<!-- correct: inline the canonical asset -->
<svg viewBox="0 0 220 240" …>
  <!-- paths copied verbatim from cocoindex.svg -->
</svg>

<!-- wrong: redrawn / approximate paths -->
<circle cx="50" cy="62" r="42" />  # do not
  • Use the SVG verbatim — do not edit colours, leaf shape, or eye geometry.
  • Render at 96–140 px wide on the 1200 × 628 canvas; the viewBox is 220 × 240.
  • Place fully inside the 88% safe area, including any speech bubble, halo, or tail.
  • Never apply filter, drop-shadow, or hue rotation. Stroke + fill contrast only.
03 — Color

Cream paper. Coral accent. Maroon text.

The canvas is always --cream. Italic emphasis in the headline always uses --coral. Code blocks sit on --maroon-ink with the canonical Shiki-mirrored token colors.

--cream #FCF3D8 · canvas --maroon-ink #2A121B · headline + code bg --maroon #532638 · deck --coral #BE5133 · italic emphasis + eyebrow --peach #E59A63 · accent + fills --palm #27E62B · success / "after" highlight --pink #FB6A76 · warning / "before" tag

No drop shadows. Use stroke + fill contrast for elevation. Never gradient-shade for depth.

04 — Rules

Do & don't.

Six rules. They cover 95% of decisions; the seventh judgement-call should always favor reading clarity over decoration.

DO · headline
One italic phrase, one period. The reader should grasp the post in one glance.
DO · safe area
Keep title + mascot inside the center 88%. Twitter crops to 2:1.
DO · canonical mascot
If Coco shows up, inline /design_guidelines/assets/cocoindex.svg verbatim — never redraw, never substitute.
DON'T · drop shadow
No box-shadow for depth. Use stroke + fill contrast instead.
DON'T · redrawn mascot
No hand-redrawn coconuts, no approximate path data, no AI-generated photo-real imagery. Canonical SVG only.
DON'T · 16:9
16:9 (1.78:1) gets letter-boxed by every social preview. Use 1.91:1 even if it feels wide.
