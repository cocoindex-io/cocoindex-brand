Page guideline · Color

Four pairings, one warm palette.

Warm maroon ink on cream paper is the baseline. Coral, peach, and pink carry emphasis. Palm is reserved for "live" signals. This page documents which colors to stack on which — the combinations that already work on every CocoIndex page, distilled.

01 — Palette

Twelve swatches. Warm by default.

Every color is warm — no cool grays, no pure white, no pure black. The cream-and-maroon foundation is the voice; the four accents are how it sings.

03 — Accents

Italic em, one color at a time.

Every serif hero has one italic word. That word gets an accent color — which one depends on the surrounding canvas. On light bg, coral is default. On dark bg, peach takes over. Pink and palm-ink are the rare spices.

04 — Code

Two surfaces, never mixed.

Inline code sits inside prose — it gets a soft coral wash so it tags a word without breaking reading rhythm. Fenced blocks stand alone — they get the maroon-ink panel, the darkest surface in the system.

Do

Inline code — coral wash + berry ink.

A paragraph mentioning flow_builder.add_source stays readable. Background is rgba(190,81,51,0.12), text is var(--berry), font 13px mono, 1 / 6 padding, 3px radius. Used in prose on the blog, docs, and these guideline pages.

Do

Block code — maroon-ink panel + cream ink.

Every <pre>, every .tokens demo, every Shiki-rendered snippet lands on var(--maroon-ink) (#2A121B). Cream text, peach/pink/palm syntax tokens. One dark surface for all fenced code, site-wide.

Don't

Don't put inline chips on maroon-ink.

A maroon-ink pill inside flowing prose punches a black hole. The coral wash reads as emphasis; the dark panel reads as a separate object — reserve it for standalone blocks, never mid-sentence.

Don't

Don't wash block code with coral.

The translucent coral is tuned for 1–3 tokens inside a paragraph. At 10+ lines it flattens syntax colors and fails contrast. Block code gets the maroon-ink anchor — always.

05 — Tokens

Drop these into :root.

The full CSS custom-property set for color. Reference these tokens — never the raw hex — so future palette shifts stay one-file changes.

/* foundation — canvas + ink */
--paper:       #FBF6E8;  /* default page bg */
--cream:       #FCF3D8;  /* filled section bg */
--cream-soft:  #F6F4E9;  /* muted card bg */
--maroon:      #532638;  /* brand ink, dark fill */
--maroon-deep: #401E2B;  /* maroon hover */
--maroon-ink:  #2A121B;  /* body text, darkest bg */

/* accents */
--coral: #BE5133;  /* primary accent, CTA, em on light */
--peach: #E59A63;  /* secondary accent, em on dark */
--pink:  #FB6A76;  /* spot accent, playful */
--berry: #6A1E23;  /* inline code text, deep accent */

/* signals */
--palm:     #27E62B;  /* live on dark bg */
--palm-ink: #16A534;  /* live on light bg (a11y) */

/* semantic — borders + muted text */
--rule:        rgba(42,18,27,0.14);  /* subtle divider */
--rule-strong: rgba(42,18,27,0.35);  /* strong divider, outlines */
--muted:       rgba(42,18,27,0.58);  /* secondary text */
