Page guideline · Admonitions

Four calls, one voice.

An admonition lifts a sentence out of the prose — a note, a prerequisite, a tip, or a warning. CocoIndex uses four variants, all on the same cream card with a hairline border and a circular icon badge. Tint and icon colour carry the signal; the typography stays identical so a doc page with three admonitions still reads as one calm system.

01 — Variants

Note · info · tip · warn.

Pick by intent. note is the quietest — a plain cream card with a berry disc, for a side-remark. info peach-washes the card to mark setup or a prerequisite. tip adds a palm wash and a check mark for a helpful, non-blocking suggestion. warn pink-washes for things that might break.

i

.co.note Side-remark or soft clarification. Berry icon; no tint on the card.

.co.info Setup, context, or prerequisite. Peach-washed cream with a coral hairline.

02 — Anatomy

Cream card, hairline border.

Every variant shares the same skeleton: 8px radius, 1px border, 14/16 padding, 14px gap between the icon disc and the body. The icon sits on a 24px circle; the label is 11px UPPERCASE mono followed by a single body line at 13.5px.

03 — Rules

Intent over decoration.

Admonitions are loud by design — every use trains readers to stop and look. Under-use them, pick the right variant, and let the default paragraph do the quiet work.

Do

One admonition per idea.

If a page has three warnings in a row, collapse them into one. The first is a signal; the third is wallpaper. Readers start skipping past them.

Don't

Don't use warn for best practices.

Warn is reserved for things that can actually break: data loss, non-reversible state, production-only footguns. "Prefer X over Y" is a tip.

Do

Lead with the consequence.

"This drops the index and rebuilds from scratch" beats "Be careful with this command." The label already says it's a warning; use the body for the specifics.

Don't

Don't invent a fifth variant.

No "success", no "danger", no custom colour. Four variants cover every case the docs need; a rogue variant breaks the visual rhythm of the page.

Do

Keep the label single-word when you can.

"Note · Prerequisite · Tip · Warning" is the short list. Custom labels are allowed ("Known issue", "Enterprise only") — keep them under three words.

Don't

Don't stack code blocks inside.

A fenced code block inside an admonition fights the wash for visual weight. Put the code after the admonition and let the body reference it in prose.

04 — Where it lives

Docs & blog prose.

Admonitions ship inside long-form content. Render them from the site's MDX admonition plugin; don't hand-write them in page templates.

Variant MDX shorthand Location
iNote :::note cocoindex/docs/src/content/docs/**/*.md
iPrerequisite :::info cocoindex/docs/src/content/docs/getting-started/*

05 — Cheat sheet

Copy-paste starts.

Markup for each variant. The .co shorthand stays in the design system; production markdown pipelines lower :::note / :::tip / etc. to the same structure.

<!-- Default · neutral annotation -->
<div class="co">
  <div class="ico">i</div>
  <div class="body"><b>Note</b>Plain-cream annotation.</div>
</div>

<!-- Note · berry icon, no tint -->
<div class="co note">
  <div class="ico">i</div>
  <div class="body"><b>Note</b>The quietest level.</div>
</div>

<!-- Info · peach-washed, setup / prerequisite -->
<div class="co info">
  <div class="ico">i</div>
  <div class="body"><b>Prerequisite</b>Start here before running the flow.</div>
</div>

<!-- Tip · palm-washed, non-blocking suggestion -->
<div class="co tip">
  <div class="ico">✓</div>
  <div class="body"><b>Tip</b>Use --dry-run to preview the diff first.</div>
</div>

<!-- Warn · pink-washed, might break -->
<div class="co warn">
  <div class="ico">!</div>
  <div class="body"><b>Warning</b>This rebuilds the index from scratch.</div>
</div>
