Page guideline · Buttons

One base, six variants.

A single .btn class carries the structural shape. Variants, sizes, and shape modifiers compose on top. Focus ring and disabled state are built in. Every token resolves from the brand palette.

01 — Variants

Six shapes of intent.

Use btn-coral for the single most important action on a view. Use btn-primary (maroon) for secondary emphasis. Everything else de-escalates from there.

02 — Sizes

Small · medium · large · extra-large.

Four real steps, one type + padding rhythm. btn-sm (12px) for inline actions and card footers · default md (14px) is the workhorse · btn-lg (16px) for prominent secondary actions · btn-xl (18px) is the hero scale.

03 — Shape

Pill, or rounded.

Pill (999px) is the default — brand-warm. Use btn-rounded (8px) inside rectangular layouts like data cards or pager tiles.

04 — Icons

Icon + label, or icon-only.

Icons live inside the button as children — gap handles spacing. For icon-only buttons, always include aria-label.

05 — Social

Hairline square, coral on hover.

LinkedIn, X, GitHub — the three canonical social glyphs. Compose btn btn-social. 32px square, 6px radius, barely-there rule border, maroon icon at rest; border and icon recolor to coral on hover. No fill. This is the exact pattern from the blog post author card — keep it consistent across the site.

32 × 32 · 6px radius — one size, everywhere
Do

Use one size — 32px.

Footer, blog post author card, share rows, author hero. Single token, single rhythm across the whole site.

Do

Recolor, don't fill, on hover.

Border and icon shift to coral together. No background change — keeps the surface quiet even when the pointer is over the glyph.

Don't

Don't use btn-outline for socials.

btn-outline has a heavy maroon border and fills maroon on hover — too loud for a byline. Use btn-social.

Don't

Don't use circles or platform colors.

Square + 6px radius is the house style. Icons inherit currentColor — never paint them LinkedIn blue, X black, or GitHub's octocat. The system owns the palette.

07 — Rules

Hierarchy rules.

One primary action per view. The rest de-escalate. Mono chips stay short.

Do

One coral CTA per view.

Pair the coral primary with an outline. Scannable hierarchy — the user knows exactly where to click.

Start indexing Read the docs
Don't

Don't stack three fills.

Coral + maroon + another fill means no primary. Every action competes and none wins.

Start indexing Read the docs Star us
Do

Mono chips for utility actions.

Short labels — 1–3 words. "Edit this page", "View source", "Copy", "Share".

Edit this page
Don't

Don't use mono for long labels.

Uppercase + tracking kills scanability past ~18 characters. Use btn-outline instead.

08 — Cheat sheet

Class combinations.

Copy-paste starting points. Compose variant + size + shape + icon as needed.

<!-- Primary CTA -->
<a class="btn btn-coral">Start indexing</a>

<!-- Hero CTA -->
<a class="btn btn-primary btn-lg">Read the docs</a>

<!-- Inline meta action -->
<a class="btn btn-mono btn-sm">View source</a>

<!-- Icon-only, circular -->
<button class="btn btn-outline btn-icon btn-icon-circle"
        aria-label="Next">…</button>

<!-- Social icon — LinkedIn / X / GitHub -->
<a class="btn btn-social"
   href="https://github.com/cocoindex-io/cocoindex"
   aria-label="GitHub"><svg>…</svg></a>

<!-- Disabled -->
<button class="btn btn-coral" disabled>Submitting…</button>
