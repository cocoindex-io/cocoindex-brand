Page guideline · Tags

Four roles, one voice.

A tag is a small label that classifies, filters, or signals state. CocoIndex uses four distinct variants — all share the same typographic voice (mono, UPPERCASE, 0.14em tracking), but each reserves a specific role so the reader can tell taxonomy from status at a glance. Never invent a fifth.

01 — Variants

Pill · category · overlay · status.

Pick by role, not by aesthetics. Clickable taxonomy uses t-pill. The one hero kicker per post uses t-category. Labels on dark media use t-overlay. State flags (Enterprise, Beta, Live) use t-status.

Knowledge graph Incremental Postgres
t-pill Clickable taxonomy. Filter links under blog posts, author pages, tag archives.
Engineering Release notes
t-category Filled coral pill. One per hero. Used only at the top of a blog post to identify its category.
Codebase · real-time Meetings · KG
t-overlay Backdrop-blurred cream-on-cream. Only on dark media — thumbnail corners, hero artwork overlays.
Live Enterprise
t-status · t-status-lg Warm pill, pulsing green dot. Large (8px dot) for hero eyebrows like the home page "Live". Default (7px dot) for inline flags like "Enterprise".
02 — Anatomy

Mono, tight, UPPERCASE.

Every variant shares the same type voice so a row of mixed tags still reads as one system. Only the chrome (fill, border, backdrop) and any leading ornament differs.

Tag label 10px mono · 0.14em · UPPERCASE · pill (999px) · 3 × 9 padding · 1px rule border
Tag label 11px mono · 0.14em · coral fill · 5 × 10 padding · no border
Tag label 10px mono · cream @ 12% · backdrop-blur(6px) · dark surface required
Live hero · 8px dot · 4→8px pulsing glow · 6/14/6/8 padding
Enterprise inline · 7px dot · 3→6px pulsing glow · 5/14/5/10 padding
03 — Density

Cluster with 6–8px gap.

Tags live in flex-wrap rows. Never horizontal-scroll a tag list — overflow hides content and hurts SEO. Cap labels at ~18 characters; longer strings belong in a title, not a tag.

RAG Embeddings Postgres Qdrant Knowledge graph Realtime Incremental Python Rust
good — flex-wrap, 8px gap, no truncation
04 — Rules

Role over look.

Tags are tiny; the wrong variant is easy to miss. These rules keep roles legible and accessible.

Do

Pulse means live.

The green-dot pulse is a semantic signal: something is on, running, or available right now. Reserve it for "Live", "Enterprise", "Beta-active" — never decorative.

Don't

Don't pulse a non-live label.

If the underlying thing isn't actually on/available, use a plain t-pill instead. Motion without meaning trains users to ignore it — the next real "Live" gets missed too.

Do

One category per post.

The filled coral t-category is the hero's single emphasized kicker. Use t-pill for the rest of the post's tags underneath.

Don't

Don't use t-overlay on light surfaces.

Backdrop-blur on cream-soft is invisible. Overlay tags belong on dark media only — thumbnails, hero art, video covers.

Do

Keep labels under ~18 characters.

"Knowledge graph" is fine. "Incremental knowledge graph indexing pipeline" is a title, not a tag — UPPERCASE 0.14em past that length becomes unreadable.

Don't

Don't invent rainbow categories.

One maroon-on-cream pill for everything; let content do the coloring. Platform or product-family colors here just add visual noise without helping classification.

05 — Where it lives

Every tag on the site.

If you're adding a new tag, start from an existing one — the class, the file, the pattern. Don't hand-roll chrome.

06 — Cheat sheet

Copy-paste starts.

Class names below are the guideline shorthand (t-*). Production uses the scoped classes listed in § 05.

<!-- Clickable taxonomy tag -->
<a class="t-pill" href="/blogs/tags/rag">RAG</a>

<!-- Filled category — one per hero -->
<span class="t-category">Engineering</span>

<!-- Overlay on dark media (backdrop-blur) -->
<span class="t-overlay">Codebase · real-time</span>

<!-- Status · Enterprise (7px pulsing dot, inline) -->
<span class="t-status">
  <span class="dt"></span>Enterprise
</span>

<!-- Status · Live (8px pulsing dot, hero eyebrow) -->
<span class="t-status t-status-lg">
  <span class="dt"></span>Live
</span>
