Page guideline · Hamburger / TOC

One sheet, two jobs.

Phones don't have rails. The hamburger inherits both navigation jobs the desktop docs split across two columns — site-wide pages on one tab, the current page's headings on the other. The GitHub star pill stays parked next to the burger, because the social-proof signal shouldn't disappear with the rails. One trigger, one drawer, one accent at a time — tap to open, scrim-tap to dismiss, ⌘K still works for power users.

01 — The experience

Tap the burger.

A right-anchored sheet slides in over a maroon scrim. The default tab is On this page because phone readers are usually mid-page; one tap to scrub headings. Switch tabs to browse the full sidebar tree without leaving the sheet.

Live preview.

The phone frame below is real markup — the burger toggles the sheet, the scrim closes it, and arrow keys move focus through the active list. Resize this window to see why phones need this: the desktop rails simply don't fit.

02 — Anatomy

Trigger, scrim, sheet, tabs.

Five regions, only one of which is visible at rest. The trigger morphs into a close-X on open, so the user always sees a single "switch state" affordance — never two competing controls.

1Topbar · burger
5Doc (under scrim)
2Sheet
3Tabs

1Burger — 32×32 ghost glyph. Three 1.4px bars in --muted, no border, no fill. The square only materializes on hover or .open; on open, the top + bottom bars rotate into a coral X and the middle fades. The bars *are* the affordance — boxing them at rest would force a third equal-weight control into a 56px-tall topbar.
★Star pill — borderless at rest. Just a star glyph + count in --muted; the pill chrome only fades in on hover/focus. Same fetch as the homepage .stars (cached in sessionStorage for an hour, falls back to — while loading and on offline) so the number agrees across surfaces — but visually it sits below the burger, not beside it as a peer.
2Sheet — right-anchored, min(86%, 320px) wide, full-height below the system status bar. Slides in over 220ms with a cubic-bezier(.2,.8,.2,1) ease. Hairline left border, no shadow.
3Tabs — segmented control, two slots: On this page (default) and Pages. Inset cream on cream-mix track; the selected slot wears a 1px --rule stroke instead of a fill swap, so the difference reads quietly.
4List — 44px-min rows. H2s carry a numbered glyph identical to the desktop Toc; H3s indent and drop the glyph. Active row is the only coral surface in the sheet.
5Scrim — 38% maroon-ink wash, fades in alongside the sheet. Tapping it dismisses. Doc behind keeps its scroll position so the user re-enters where they left.

04 — Rules

One trigger, one accent, one path back.

The hamburger is the only off-canvas surface in the docs. Keep it predictable — the same gesture opens and closes; the same scrim always dismisses; the same arrow keys always step through rows.

Do

Default to On this page.

Most phone visits arrive mid-doc from search or a shared link. Showing the heading list first cuts the scroll-back to nothing — the user lands, taps, jumps.

Don't

Don't add a third tab.

Search belongs in the topbar (⌘K), not in the sheet. A tab for "Search", "Settings", or "Account" turns the sheet into an app drawer — readers lose the model.

Do

Hide it on desktop.

The sheet is a phone-only fallback. @media (min-width: 880px) brings the rails back; the burger and sheet drop to display: none. No "menu button" on desktop.

Don't

Don't shadow the sheet.

Elevation reads from the maroon scrim, not from blur. A drop shadow on cream over cream looks dirty and fights the rest of the design system. Hairline + scrim is the answer.

Do

Persist the active row.

Scroll-spy still runs while the sheet is open. When the user closes and reopens, the highlight reflects where they actually are on the page — not where they last tapped.

Don't

Don't trap scroll.

The doc behind the scrim should not scroll while the sheet is open (toggle overflow: hidden on body). When the sheet closes, restore the user's prior scroll position — never reset to top.

05 — Cheat sheet

Drop-in markup.

CSS owns morphing the burger and sliding the sheet. JS owns three things: open/close, tab switching, and Esc-to-dismiss. Everything else is declarative.

// Controller — open/close, tabs, Esc, scroll-lock
const burger = document.getElementById('phBurger');
const sheet  = document.getElementById('phSheet');
const scrim  = document.getElementById('phScrim');

function setOpen(open){
  burger.classList.toggle('open', open);
  sheet.classList.toggle('open', open);
  scrim.classList.toggle('open', open);
  burger.setAttribute('aria-expanded', open);
  document.body.style.overflow = open ? 'hidden' : '';
  if(open) sheet.querySelector('[role="tab"][aria-selected="true"]').focus();
}
burger.addEventListener('click', () => setOpen(!sheet.classList.contains('open')));
scrim .addEventListener('click', () => setOpen(false));
document.addEventListener('keydown', e => { if(e.key === 'Escape') setOpen(false); });
