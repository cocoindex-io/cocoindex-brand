Page guideline · Docs link card

Send the reader to the docs.

A docs link card lifts an outbound reference (usually cocoindex.io/docs/...) out of the prose and gives it a clear hand-off — eyebrow, italic-accented title, one-line description, arrow. One variant only; one card per resource. Hover swaps in the dotted brand pattern that echoes the end-of-post CTA strip — the card stays anchored in the reading flow, never translates.

01 — Anatomy

Icon disc · eyebrow · serif title · description · arrow.

All five parts are required. The serif title carries a single <em> accent (the load-bearing word for the link target). The description is one sentence of plain prose with optional code chips, max ~22 words.

02 — States

Resting and hover.

Resting: cream fill, hairline rule. Hover: a peach dot pattern fades in on the right side of the card (masked off on the left so the prose-side stays clean), the border deepens to maroon, and the arrow nudges 3 px right while recolouring to palm-ink (sage-sprout green). Same opacity 0 → 1 transition the home page .cp-tile uses. No translate, no fill change, no jump.

03 — Usage

Where it goes, where it doesn't.

Reach for it when the next reasonable thing a reader might do is leave the post and read reference material on the same topic. Cap one or two per long-form post — more than that and it stops feeling editorial.

Do

Place after a section that introduces a feature.

Right after the prose explains what the connector does is the highest-intent moment to offer reference docs. The card sits in the natural pause between sections.

Do

Use the italic em for the load-bearing word.

"Kafka target connector." or "Programming guide." The accent makes the title scannable without bolding the whole line.

Don't

Don't use as the post's closing CTA.

The end-of-post CTA strip (`.cta` in the blog-post template) is maroon-bg with a coral pill button — it does that job. The docs link card is mid-article, cream, quieter.

Don't

Don't lift on hover.

The card lives inline in body copy. Translates and shadow lifts make it bounce as the reader scrolls past — the dotted pattern + arrow nudge is the whole interaction.

04 — API

Astro component props.

Wrapper component lives at src/components/DocsLinkCard.astro. Import it from an MDX post and pass props. title and description accept HTML so authors can include <em> and <code> inline.

PropTypeNotes
href string · required Destination URL — typically cocoindex.io/docs/... or another canonical reference page.
title string · required HTML-permitted. Use one <em>...</em> for the accent word; end with a period.
description string · optional HTML-permitted. One sentence, ~22 words max. Wrap identifiers in <code>.
eyebrow string · default "Documentation" The kicker. Override only when pointing somewhere that isn't docs (e.g. "Reference", "Guide").
// MDX usage import DocsLinkCard from '../../../components/DocsLinkCard.astro'; <DocsLinkCard href="https://cocoindex.io/docs/connectors/kafka/" title="Kafka <em>target</em> connector." description="Reference docs for CocoIndex's Kafka target — config options, <code>declare_target_state</code> semantics, key/value mapping, and topic management." />
